### Skip strategy (xunit v3)

xunit v3 has **three** skip mechanisms — use **all three** per fixture:

1. **Class-level trait for CI filter**: `[Trait("RequiresKey", "OPENAI_API_KEY")]` on the fixture class. CI can `--filter "Trait!=RequiresKey"` to run fixture-free.
2. **Runtime skip inside `GetChatClient()`**: `if (Environment.GetEnvironmentVariable("OPENAI_API_KEY") is null) Assert.Skip("OPENAI_API_KEY not set");` — xunit v3 `Assert.Skip` throws `SkipException` which xunit recognizes.
3. **Async probe for Ollama only**: before returning the client, `using var probe = new HttpClient(); try { await probe.GetAsync(new Uri(endpoint + "/api/version"), cts.Token); } catch { Assert.Skip("Ollama daemon unreachable"); }` — daemon-down is a runtime skip, not a missing-key skip.

Do NOT use `[Fact(Skip = "...")]` — that's unconditional. Runtime skip is mandatory.
